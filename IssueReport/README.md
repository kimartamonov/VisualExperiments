# IssueReport

Эта папка хранит артефакты завершения по каждому реализованному Issue.

## Naming Convention

- `IssueReport/{ISSUE_ID}_Implementation_Report.md`
- `IssueReport/{ISSUE_ID}_Technical_Documentation.md`

## Expected Content

### Implementation Report

- что было реализовано;
- какие файлы изменены;
- какие acceptance criteria закрыты;
- какие тесты реально выполнены;
- какие ограничения остались;
- что теперь разблокировано.

### Technical Documentation

- назначение решения;
- затронутая архитектура;
- контракты данных;
- ключевая логика;
- ограничения;
- интеграционные точки;
- стабильная опора для следующего Issue.
